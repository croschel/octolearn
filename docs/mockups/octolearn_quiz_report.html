Learning resume

Collapse

Lambda cold starts occur when AWS must initialize a new execution environment for a function that hasn't been invoked recently — or at all. This involves downloading your code, bootstrapping the runtime, and running initialization code outside your handler. Cold starts add latency ranging from a few hundred milliseconds to several seconds depending on runtime and package size.

Execution context reuse is the mechanism that makes warm starts fast. When a Lambda function stays warm, AWS reuses the same container, keeping the initialized runtime and any global-scope resources (DB connections, SDK clients) alive between invocations. This is why initializing expensive resources outside your handler is a best practice.

Lambda layers are ZIP archives attached to your function's execution environment — they're the right tool for sharing code libraries, custom runtimes, or binary dependencies across multiple functions. Lambda extensions are separate processes that run alongside your function and integrate with the Lambda lifecycle API, making them ideal for observability agents, secrets managers, and security tooling.
