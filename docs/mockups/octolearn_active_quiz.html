Explain the difference between Lambda layers and Lambda extensions, and when you'd use each.

212 characters

🐙 Full explanation — after 3 attempts

Lambda Layers package code and data that multiple functions share (e.g. shared utilities, runtime binaries, ML models). They're added to the function's execution environment. Lambda Extensions, on the other hand, run as separate processes alongside your handler — they integrate with the Lambda runtime API to receive lifecycle hooks. Use layers for code/dependency sharing; use extensions for telemetry, secrets injection, or security tooling.
